# macOS 装机助手真实项目图设计

## 目标

用用户提供的 macOS 装机助手界面截图替换该项目当前的占位封面，并在中英文项目详情页展示同一张真实主图。图片必须完整可见，不裁切界面内容。

## 成功标准

- 中文和英文的 macOS 装机助手项目卡片都使用 `/images/macos-setup-assistant.png`。
- 中文和英文项目详情页都在摘要后、正文前显示该图片。
- 卡片和详情页在桌面及移动端均保持 16:9 展示区域，以黑色为底并使用 `object-fit: contain`，不裁切原图。
- API Pulse 和其他内容继续使用现有封面及展示方式。
- 不修改截图中的文字或界面，不添加未经证实的项目成果，不扩大到其他视觉改版。
- 单元检查、Astro 检查、构建和端到端测试全部通过。

## 资源处理

将用户提供的原始 PNG 按原像素复制为：

`public/images/macos-setup-assistant.png`

不压缩、不生成新图、不进行裁切或内容编辑。复制后以 SHA-256 对比源文件与目标文件，确认像素资源未改变。

## 内容模型

在项目内容 schema 中增加：

```yaml
coverFit: cover | contain
```

该字段默认值为 `cover`，因此现有项目行为不变。macOS 装机助手的中英文内容条目显式设置 `coverFit: contain`，并把 `cover` 指向新图片。

采用该字段而非在组件中判断项目名称或图片路径，避免展示规则与某个项目硬编码绑定；只增加当前需求所需的两种取值，不引入更多配置。

## 页面展示

### 项目卡片

`ProjectCard` 根据 `coverFit` 添加对应的图片展示类。默认项目继续铺满并裁切；macOS 装机助手图片在现有 16:9 卡片区域内完整缩放，空余区域显示黑色。

### 项目详情页

`ContentLayout` 增加可选的封面参数，在摘要后、正文前渲染主图。只有项目内容页传入该参数，文章页面保持不变。

中英文项目路由仅在项目封面不是默认占位图时传入封面参数。这样 macOS 装机助手显示真实主图，API Pulse 详情页不会新增占位主图。主图沿用内容条目的标题生成本地化替代文本，并使用异步解码；首屏详情主图不强制懒加载。

## 样式与响应式

- 卡片和详情主图外框使用 `aspect-ratio: 16 / 9`。
- `contain` 模式使用黑色背景和 `object-fit: contain`。
- 图片宽度不超过容器，移动端不产生横向滚动。
- 不改变站点现有卡片尺寸、排版、色彩变量或其他项目图片规则。

## 验证方案

按测试驱动方式先增加失败测试，再实现功能：

1. 验证中英文 macOS 装机助手卡片引用真实图片并计算为 `object-fit: contain`。
2. 验证中英文项目详情页包含真实主图，替代文本正确，计算样式为 `contain`。
3. 验证 API Pulse 仍使用现有占位封面，且详情页不新增占位主图。
4. 在桌面和移动端视口检查图片容器比例、无裁切规则及无横向溢出。
5. 运行项目完整验证命令、依赖审计和生产构建扫描。

## 发布

本地验证通过后提交实现并合入 `main`，推送 GitHub。等待 GitHub Pages 工作流完成，再检查中英文首页、两个项目详情页和图片资源的线上响应与呈现。
